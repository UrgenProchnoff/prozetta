# prozetta

*Проза + Розетта* — конвейер на LLM для **художественного перевода**, который держит
имена и термины едиными по всей книге и сам проверяет качество.

🇬🇧 [English version](README.md)

Обычный машинный перевод обрабатывает главу в отрыве от остальных, поэтому персонаж
«Ли Вэй» через сто страниц превращается в «Ли Уэй» и «господина Ли». prozetta работает
в два этапа, чтобы этого не происходило:

1. **Подготовка контекста** — читает книгу целиком, разбивает на чанки, извлекает
   имена и термины и сводит их в единый глоссарий, который можно проверить и поправить.
2. **Умный перевод** — переводит чанк за чанком через цикл
   **перевод → проверка → решение → исправление/переперевод**, используя глоссарий
   как «истину».

Поверх того же конвейера работает веб-интерфейс для правки глоссария, мониторинга
прогресса и точечного исправления чанков.

## Предварительные требования

1. Установленный **Node.js**.
2. Установленные зависимости:
   ```bash
   npm install
   ```
3. Источник LLM — на выбор:
   - **локальный** OpenAI-совместимый сервер (llama.cpp / vLLM / LM Studio и т.п.), или
   - **облачные** провайдеры по API-ключам:
     ```bash
     export GOOGLE_API_KEY="ваш_ключ"   # Google Gemini
     export GROQ_API_KEY="ваш_ключ"     # Groq
     ```

## Конфигурация

Настройки моделей — в [src_v4/config.js](src_v4/config.js):

- **logic_model** — основная модель перевода и проверки (по умолчанию локальный сервер)
- **google_model** — Google Gemini
- **groq_model** — Groq

Там же лежат параметры конвейера (размеры чанков, число повторов и пороги проверки —
принять при оценке ≥ 9.1, доработать при ≥ 7.5, переписать ниже).

**Рекомендуемые модели.** Хорошие результаты в роли logic_model даёт
`gemma4-26b-a4b` и выше. При чанках ~1к токенов (по умолчанию) для ризонинг-моделей
рекомендуется контекст ~32к токенов — чанк, глоссарий и цепочка рассуждений/проверки
требуют запаса.

## Быстрый старт

```bash
# Этап 1 — строим глоссарий (провайдер задаётся --model, по умолчанию local)
node src_v4/main.js --stage=1 --file=txt/My_Book.txt --model=google

# Проверьте и отредактируйте My_Book_glossary.json (или через GUI), затем:

# Этап 2 — перевод (по завершении сам соберёт txt/My_Book_rus.txt)
node src_v4/main.js --stage=2 --file=txt/My_Book.txt --model=google
```

`--model` принимает `local` (по умолчанию), `google` или `groq`.

### Параметр `--file`

`--file` **обязателен на всех этапах**. Он определяет проект: исходный файл, префикс
для `<префикс>_project_state.json` и `<префикс>_glossary.json` и имя выходного файла.
Это позволяет вести **несколько книг параллельно**, каждая со своим независимым состоянием.

### Цикл проверки на Этапе 2

Каждый чанк проходит через: **перевод → проверка → решение → исправление/переперевод**.

| Результат проверки        | Действие                          |
|---------------------------|-----------------------------------|
| оценка ≥ 9.1 и `like = 1` | ✅ **Принят** — сохраняем          |
| оценка ≥ 7.5 и `like = 1` | 🔧 **Доработка** — исправляем ошибки |
| оценка < 7.5 или `like = 0` | 🔄 **Заново** — перевод с нуля    |

Максимум 10 итераций на чанк; при исчерпании попыток сохраняется лучший вариант.
Прерванный процесс продолжит с того же места при повторном запуске.

## GUI

Веб-интерфейс поверх того же конвейера (CLI продолжает работать как раньше):

```bash
npm run gui
```

Откройте `http://127.0.0.1:3457` (порт настраивается переменной `GUI_PORT`; сервер
слушает только `127.0.0.1`). Возможности:

- **Дашборд** — все проекты с прогрессом, новые книги из `txt/`.
- **Глоссарий** — табличный редактор вместо ручной правки JSON: поиск, типы, род,
  заметки, счётчик вхождений термина в чанках (0 — кандидат на удаление).
- **Монитор** — запуск/остановка этапов, живой лог, карта чанков с цветовой кодировкой.
- **Чанк** — оригинал и перевод рядом, ручное редактирование, «принять» или «сбросить»,
  полная история попыток с оценками.

## Дополнительные инструменты

```bash
# Принудительный экспорт того, что уже переведено (даже если перевод не завершён)
node src_v4/main.js --stage=export --file=txt/My_Book.txt

# Сброс Этапа 2 с сохранением результатов Этапа 1 (создаётся бэкап состояния)
node src_v4/tools/reset_to_stage1.js --file=txt/My_Book.txt
```

## Структура проекта

```
src_v4/                     # Ядро конвейера
  main.js                   # Точка входа CLI
  config.js                 # Настройки моделей и конвейера
  core/                     # llm_client, state_manager, tokenizer
  stages/                   # 01_extraction, 02_consolidation, translation_loop
  tools/reset_to_stage1.js  # Сброс переводов
  utils/                    # parsers, rate_limiter
src_gui/                    # Веб-интерфейс поверх конвейера
txt/                        # Тексты и результат (в .gitignore)
```

Файлы состояния (`*_project_state.json`, `*_glossary.json`) и папка `txt/` — в
`.gitignore`, ваши тексты и глоссарии остаются только у вас.
