# Инструкция по использованию V4 Переводчика

Этот переводчик работает в два этапа (стадии), чтобы обеспечить высокое качество перевода с сохранением контекста и авторского стиля.

## Предварительные требования

1.  Установленный **Node.js**.
2.  Установленные зависимости:
    ```bash
    npm install
    ```
3.  Запущенные локальные сервера с LLM (если используется локальный режим) или ключ API Google.

---

## Конфигурация

Настройки моделей находятся в файле `src_v4/config.js`.
*   **logic_model**: Основная модель для перевода и "мышления" (Qwen, Gemini).
*   **fast_model**: Модель для быстрых задач (пока используется мало).
*   **google_model**: Настройки для Gemini (API ключ, имя модели).

Для Google модели убедитесь, что ключ экспортирован:
```bash
export GOOGLE_API_KEY="ваш_ключ"
```

---

## Рабочий процесс (Workflow)

### Этап 1: Подготовка Контекста (Extraction & Consolidation)

На этом этапе программа читает книгу, разбивает на части и извлекает имена, термины и названия. Затем она создает единый глоссарий.

**Команда:**
```bash
node src_v4/main.js --stage=1 --file="путь/к/книге.txt"
```

**Пример:**
```bash
node src_v4/main.js --stage=1 --file="/home/user/books/my_book.txt"
```

**Результат:**
*   Создается файл `project_state.json` (прогресс).
*   Создается файл `glossary.json` (глоссарий терминов).

### ВАЖНО: Ручная правка глоссария
После завершения Этапа 1, откройте файл `glossary.json` в текстовом редакторе.
*   **Проверьте переводы**: Удалите мусор, исправьте неверные переводы имен.
*   **Сохраните файл**.
*   Именно этот глоссарий будет использоваться моделью как "истина" на этапе перевода.

---

### Этап 2: Умный Перевод (Translation Loop)

На этом этапе идет перевод каждого фрагмента. Система использует цикл (loop): переводит -> проверяет (критик) -> исправляет, пока не получится хорошо (или пока не закончатся попытки).

**Команда (Локальная модель - по умолчанию):**
```bash
node src_v4/main.js --stage=2
```

**Команда (Google Gemini):**
```bash
node src_v4/main.js --stage=2 --model=google
```

**Команда (Groq API):**
```bash
node src_v4/main.js --stage=2 --model=groq
```
*Требуется `GROQ_API_KEY`*

*Система автоматически подхватит прогресс из `project_state.json`.*

**Результат:**
*   По завершении автоматически создается файл перевода (например, `my_book_rus.txt`) рядом с оригиналом.
*   Если процесс был прерван, повторный запуск продолжит с того же места.

## Дополнительные инструменты

### Принудительный Экспорт
Если вы хотите собрать книгу до завершения всего перевода (то, что есть):
```bash
node src_v4/main.js --stage=export
```

### Сброс Перевода (Reset to Stage 1)
Если вы хотите начать перевод (Этап 2) заново (например, после сильного изменения промптов или глоссария), но не хотите заново ждать экстракцию терминов (Этап 1).

Этот скрипт удалит все переводы, но оставит исходный текст и результаты экстракции:
```bash
node src_v4/tools/reset_to_stage1.js
```
*Будет создан бэкап текущего состояния.*
